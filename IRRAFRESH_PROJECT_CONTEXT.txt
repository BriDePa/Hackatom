============================================================
ACTUALIZACION AUTORITATIVA — PHASE
============================================================

PHASE significa Post-Harvest Analysis & Simulation Engine.

PHASE es la evolucion del motor de simulacion de IRRAFRESH. Mantiene la misma
arquitectura MVP, pero refuerza el enfoque cientifico: separa parametros
biologicos internos de parametros operativos del usuario, usa perfiles
logisticos abstractos, agrega eventos dinamicos y mejora la lectura como
Decision Support System.

Cambios PHASE implementados:
- Producto nuevo: papa.
- Los parametros cientificos de tomate, fresa y papa se cargan desde datasets
  internos JSON/CSV y no se editan desde la UI.
- Las rutas reales fueron reemplazadas por perfiles logisticos abstractos:
  optimal_cold_chain, export_stress y ambient_long_distance.
- El usuario modifica parametros operativos: producto, dosis, temperatura,
  humedad, duracion, tipo de transporte, almacenamiento refrigerado y eventos.
- Eventos dinamicos agregados: falla de refrigeracion, retraso de transporte,
  exposicion termica y almacenamiento prolongado.
- Escenarios rapidos agregados: Optimal Cold Chain, Export Stress Scenario,
  Refrigeration Failure, High Humidity Storage y Long Distance Transport.
- El motor recalcula deterioro, riesgo, economia, alertas y timeline de eventos.
- La UI ahora muestra un timeline visual de eventos logisticos.

Esta seccion supersede cualquier referencia anterior a rutas reales especificas
como parte del modelo activo. Esas rutas ya no son el enfoque del sistema.

============================================================
MODELO CIENTIFICO ACTUAL
============================================================

El modelo representa tendencias relativas de deterioro post-cosecha.

Indice de calidad:
Q representa un indice compuesto simplificado de:
- frescura
- estabilidad microbiologica
- valor comercial
- integridad post-cosecha

No es una medicion de laboratorio.

Modelo base:
Q(t) = Q0 * exp(-k(t) * t)

k no es constante. El sistema usa deterioro dinamico:
k(t) = k0 + a*T + b*H - c*D + r*L

Interpretacion:
- mayor temperatura aumenta deterioro
- mayor humedad aumenta presion microbiologica
- mayor irradiacion reduce deterioro relativo
- mayor estres logistico aumenta deterioro
- una falla de cadena de frio aumenta temporalmente k(t)

Variables conceptuales:
- k0: deterioro base por alimento
- T: presion de temperatura
- H: presion de humedad
- D: dosis gamma
- L: estres logistico
- r: sensibilidad del producto al transporte

============================================================
MODELO DE RIESGO
============================================================

El sistema calcula una presion de riesgo relativa:

R = w1*T + w2*t + w3*L - w4*D

Salida visible:
- LOW / BAJO
- MEDIUM / MEDIO
- HIGH / ALTO

El sistema evita falsa precision:
- No muestra "17.382% de riesgo".
- Usa rangos, indices relativos y niveles categoricos.

============================================================
MODELO DE FALLA DE CADENA DE FRIO
============================================================

El usuario puede activar:
cold_chain_failure = true

Tambien puede definir:
cold_chain_failure_hours

Efecto:
- Durante ese evento, k(t) aumenta temporalmente.
- La curva de deterioro se recalcula.
- El riesgo relativo aumenta.
- El dashboard muestra alerta visual.
- Los puntos de la curva incluyen cold_chain_event.

Ejemplo conceptual:
"Cold chain failure during 6 hours."

============================================================
MODELO ECONOMICO
============================================================

El sistema calcula perdida estimada:

Loss = P * (1 - Q(t)/Q0)

Donde:
- P = valor estimado del lote
- Q(t) = calidad simulada en el momento logistico
- Q0 = calidad inicial

Tambien calcula:
- perdida estimada del escenario irradiado
- perdida estimada relativa del escenario no irradiado
- ahorro potencial por irradiacion

Todos estos valores se muestran como rangos.

============================================================
INCERTIDUMBRE
============================================================

El sistema comunica incertidumbre mediante:
- rangos de calidad
- rangos de vida util
- rangos de perdida economica
- bandas de incertidumbre en curva
- nivel de confianza

Niveles:
- LOW
- MEDIUM
- HIGH

En la practica actual, el sistema tiende a usar confianza media o baja
segun severidad logistica, porque el modelo es exploratorio.

============================================================
PRODUCTOS SOPORTADOS
============================================================

1. Tomate
2. Fresa

Cada producto tiene parametros independientes.
No comparten el mismo comportamiento matematico.

Tomate:
- Vida util base mayor.
- Sensibilidad respiratoria y microbiana moderada.
- Temperatura optima aproximada: 12 C.
- Humedad optima aproximada: 85%.

Fresa:
- Vida util base menor.
- Sensibilidad respiratoria y microbiana alta.
- Temperatura optima aproximada: 4 C.
- Humedad optima aproximada: 90%.

Papa:
- Vida util base mayor.
- Sensibilidad respiratoria baja.
- Alta respuesta relativa a irradiacion por control de brotacion.
- Temperatura optima aproximada: 10 C.
- Humedad optima aproximada: 92%.

============================================================
PERFILES LOGISTICOS SOPORTADOS
============================================================

No se usan rutas reales especificas en la simulacion activa. Se usan perfiles
abstractos para que el sistema sea mas flexible, general y defendible
cientificamente.

1. Optimal Cold Chain
2. Export Stress Scenario
3. Long Distance Transport

Cada perfil incluye:
- duracion estimada
- temperatura promedio
- si tiene refrigeracion
- riesgo de retraso
- estabilidad termica
- estres logistico

Tipos de transporte operativos:
- refrigerated_truck
- ambient_truck
- mixed_cargo
- air_cargo

Eventos logisticos operativos:
- cold_chain_failure
- delay_hours
- thermal_exposure_hours
- extended_storage_hours

Escenarios rapidos en UI:
- Optimal Cold Chain
- Export Stress Scenario
- Refrigeration Failure
- High Humidity Storage
- Long Distance Transport
